Documentación del Código

Este código implementa una aplicación web utilizando Flask y Flask-SocketIO para la comunicación en tiempo real entre el frontend y el backend. Además, se integra con dispositivos Arduino mediante comunicación serial, bases de datos para almacenamiento de datos, y permite ejecutar comandos CLI.

Estructura General

Importación de Módulos

El código importa los siguientes módulos:

  • Flask: Para crear la aplicación web.
  • Flask-SocketIO: Para habilitar la comunicación en tiempo real entre el servidor y los clientes.
  • threading: Para manejar tareas en segundo plano usando hilos.
  • time: Para gestionar tiempos de espera y calcular intervalos.
  • subprocess: Para ejecutar comandos en la CLI.
  • queue: Para manejar colas de datos.
  • SerialManager: Módulo personalizado para gestionar la comunicación serial con Arduino.
  • DBConnection: Módulo personalizado para interactuar con la base de datos.
  • CLIworker: Módulo personalizado para compilar y cargar código en el Arduino.

Variables Globales

El código define las siguientes variables globales:

  • JsonVar: Diccionario que contiene los valores de configuración para el Arduino (PID, setpoint, modo, botón, etc.).
  • serial_lock: Lock para evitar conflictos al acceder al puerto serial desde múltiples hilos.
  • currentMode: Variable que almacena el modo actual del sistema.
  • temporalList: Lista temporal para almacenar datos.
  • last_serial_time: Registro del último tiempo de comunicación serial.
  • intervalSer: Intervalo de tiempo entre comunicaciones seriales.
  • ser: Objeto que representa la conexión serial.
  • threads_started: Bandera para evitar iniciar múltiples hilos.
  • QUEUE_MAX: Tamaño máximo de las colas de datos.
  • DB_BATCH_SIZE: Tamaño del lote para guardar datos en la base de datos.
  • db_queue: Cola para almacenar datos antes de enviarlos a la base de datos.
  • frontend_queue: Cola para almacenar datos antes de enviarlos al frontend.
  • fqbn: Identificador del modelo de Arduino.
  • PROYECTO: Nombre del proyecto Arduino.

Rutas y Eventos

Ruta /

Descripción: Renderiza la página principal de la aplicación web.


@app.route("/")
def Rederizer():
    nombre = "HMI"
    return render_template("HMIv2.html",nombre=nombre)

Evento connect

Descripción: Se ejecuta cuando un cliente se conecta al servidor.

Evento disconnect

Descripción: Se ejecuta cuando un cliente se desconecta del servidor.

Evento command

Descripción: Maneja los comandos enviados desde el frontend.

Funciones Principales

start_background_tasks()

Descripción: Inicia los hilos en segundo plano para la ejecución de tareas concurrentes.

serial_worker()

Descripción: Lee datos del puerto serial y los coloca en las colas correspondientes.

send_data_Fronted()

Descripción: Envía datos almacenados en la cola al frontend mediante SocketIO.

reconnect_serial()

Descripción: Reconecta el puerto serial en caso de fallos.

WatchDog()

Descripción: Monitorea el tiempo de respuesta del sistema y reconecta el puerto serial si el tiempo supera el límite definido.

commandFilter()

Descripción: Filtra y ejecuta comandos según su tipo.

UpdateJsonArduino()

Descripción: Envía el diccionario JsonVar al Arduino mediante comunicación serial.

DbWorker()

Descripción: Guarda los datos de posición en la base de datos.

Resumen

Este código realiza las siguientes tareas:

  • Frontend: Renderiza una interfaz web y gestiona la comunicación en tiempo real con el cliente.
  • Backend: Procesa comandos y datos enviados por el cliente.
  • Serial: Maneja la comunicación con dispositivos Arduino.
  • Base de Datos: Almacena datos de posición en una base de datos.
  • CLI: Compila y carga código en el Arduino.
